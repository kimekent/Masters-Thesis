Q: Ich habe auf dieser Seite einen Image Teaser eingebaut und dann noch verlinkt mit einem A3 JPG. Leider geht das Pop-Up nicht auf.
Was habe ich vergessen? Konnte es in Web-Help nicht rausfinden.

A: Um ein Bild einzufügen, das beim Draufklicken grösser dargestellt wird, kann ein manueller Teaser oder eine Bildergeschichte (mehrere Bilder ) eingefügt werden. Beim Bearbeiten muss die Checkbox "Link (nur Bilder via Media Link) in Popup vergrössern" ausgewählt werden. Das vergrösserte Bild muss als Link ausgewählt werden.
Falls der Haken bei "Link (nur Bilder via Media Link) in Popup vergrössern" gesetzt wurde aber das Bild beim Draufklicken nicht grösser dargestellt wird, liegt es wohl daran, dass das Bild noch nicht in der grossen Version hochgeladen wurde. Damit dar Popup funktioniert, muss das Bild 1x in der vorgegebenen Grösse für den Teaseroder die Bildergeschichte hochladen werden und 1x in der Grösse, wie es vergrössert dargestellt werden soll. Wenn der Teaser dann bearbeitet wird, kann im Bereich  "Link" durch die Funktion  "Insert Media Link" das grössere Bild und die Checkbox "Link in Popup vergrössern" ausgewählt werden. Dadurch wird das grössere Bild beim Klick auf das Teaserbild angezeigt.

Q: Ich habe Änderungen an der Seite vorgenommen und gespeichert, aber sie ist immer noch nicht online. Was muss ich tun?
A: Wenn eine Seite nicht erscheint kann das an den Publish Einstellungen liegen: 
Im Experience Editor kann auf der Seite über die den Befehl "Campus > Publish Options" im Ribbon-Menü kontrolliert werden, ob und wann Inhalte publiziert werden.
Im Feld "Publish" definiert die Redaktion, wann ein nicht veröffentlichter Inhalt publiziert wird. Achtung: Die Seite wird immer auf den nächstfolgenden Veröffentlichungszeitpunkt live geschalten (entweder um 12:05 oder 00:05).

Im Feld "Unpublish" definiert die Redaktion, wann ein veröffentlichter Inhalt zurückgezogen wird.

Ist die Checkbox "Never publish" angekreuzt, wird ein Inhalt nicht publiziert.
Publish / Unpublish kann auch bei einzelnen Teaser oder auf Files in der Media Library angewandt werden.

Publish / Unpublish kann auch bei einzelnen Teaser oder auf Files angewandt werden

Ein weiterer Grun könnte an den Sprachversionen liegen. Seiten wie auch Seitenelemente müssen immer auf deutsch vorhanden sein. Wenn ein Teaser in der deutschen Version gelöscht wird, wird er auch in der englischen gelöscht. Es gilt die Regel, keine Inhalte nur in englisch, da dies fuer SEO schädlich ist.
 
Q: 
 Wenn ich in verschiedenen Containern eigene Bilder einfuegen möchte, sprich Bilder von meine Computer, wie muss ich das machen? Ich dachte man muesse auf upload und hätte dann Zugriff. Die Bilder habe ich bereits vorbereitet und ans richtige Format angepasst.
A: [Hier](https://applsupport.hslu.ch/webhelp/index.html?hm_upload_files.htm) steht beschrieben, wie du Bilder in die Media Library lädst: 1. Im Ribbon-Menü Campus > Media Library wählen.

2.In der Media Library im Item Tree der entsprechenden Ordner für deinen Speicherort auswählen. Wenn kein Ordner ausgewählt wird, erscheint eine Fehlermeldung, die das Speichern der Datei oder des Bildes verhindert.
  
3.Im Reiter Folder auf das Upload-Files-Icon klicken.
 
4.Gewünschte Dateien im Explorer-Fenster auswählen, z.B. Bilder und auföffnen klicken. Die Bilder werden hoch geladen.
 
Hinweis: Mit gedrückter Ctrl-Taste können mehrere Files ausgewählt werden.

5.Die Files wurden in den ausgewählten Ordner (hier der Ordner ta) hoch geladen. Im rechten Bereich Media werden die Bildinhalte mit einem Warnhinweis dargestellt. Dies, weil bei den Bilder noch die Alt-Texte fehlen und ausgefüllt werden müssen.
 

6.Links im Item Tree auf das jeweilige Bild klicken und anschliessend im Content Bereich auf der rechten Seite den Alt-Text eingeben. Dieser Text sollte das Bild beschreiben. Die Eingaben speichern.
 Danach kannst du beim entsprechenden Teaser das gewuenschte Bild einfuegen.
 Du kannst ein Bild auch direkt ueber den Teaser uploaden. Dazu auf das "bearbeiten"-Icon des Teasers klicken, beim Feld "Bild" auf Browse klicken und dann auf Upload.

Falsch: "Wir veröffentlichen eine neue Studie."
Spärlich einsetzen: "Lesen Sie die neue Studie des ITW!"

Alt-Texte machen Bilder allen zugänglich
Nicht alle können Bilder sehen. Der Alt-Text (auch Alt-Tag genannt) ersetzt in unserem Fall die Bildunterschrift (‚Caption‘) und stellt sicher, dass ein Bild via Screenreader allen zugänglich gemacht wird. Ein Screenreader ist ein Programm, welches Text für Menschen mit eingeschränkter optischer Wahrnehmung ‚vorliest‘.
Ausserdem beeinflussen die Alt-Texte die Suchmaschinen-Platzierung positiv (SEO-Ratings).
 
Jedes auf der Website dargestellte Bild muss deshalb zwingend mit einem Alt-Text versehen werden.
 
-Alt-Text: Hier beschreiben, was das Bild darstellt. Browser stellen den Alt-Text dar, wenn das Bild nicht geladen werden kann und halten den Beschrieb für Screenreader bereit.
 
         Beispiel: „Skyline von Chicago, gesehen von den Indiana Dunes.“
 
·Copyright-Informationen: Als Klammerbemerkung in den Alt-Text integrieren.
 
         Beispiel: „Skyline von Chicago (Copyright Tonietta Muster).“

RESPONSE:

Die Fehlermeldung beim Abspeichern von Bildern kann verschiedene Ursachen haben, die du überprüfen solltest:

1. Bildgrösse: Das Bild muss auf die richtige Grösse zugeschnitten und progressiv gespeichert werden. Überprüfe, ob die Abmessungen und die Auflösung des Bildes den Anforderungen entsprechen. Informationen dazu findest du [hier](https://applsupport.hslu.ch/webhelp/hm_photoshop.htm) und [hier](https://applsupport.hslu.ch/webhelp/hm_bilder.htm).

2. Namenskonvention: Stelle sicher, dass der Dateiname des Bildes keine Sonderzeichen oder Umlaute enthält und klar angibt, was das Bild darstellt. Weitere Informationen zur Namenskonvention findest du [hier](https://applsupport.hslu.ch/webhelp/hm_bilder.htm).

3. Alt-Text: Jedes auf der Website dargestellte Bild muss mit einem Alt-Text versehen werden, um die Barrierefreiheit zu gewährleisten. Stelle sicher, dass der Alt-Text eine präzise Beschreibung des Bildinhalts enthält. Weitere Informationen findest du [hier](https://applsupport.hslu.ch/webhelp/hm_bilder.htm).

Wenn du alle diese Punkte überprüft hast und die Fehlermeldung weiterhin erscheint, wende dich an den Support, um weitere Unterstützung zu erhalten.

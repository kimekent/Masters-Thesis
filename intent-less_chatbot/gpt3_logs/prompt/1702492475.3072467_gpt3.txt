AKTUELLE FRAGE: Wie lösche ich die englische Sprachversion eines Teasers, ohne die deutsche zu löschen?

ÄHNLICHE FRAGEN UND DAZUGEHÖRIGE ANTWORTEN:
Q: Kann ich englische Versionen (nicht übersetzt, da nicht vorgesehen) löschen  und wenn ja, wie?
 
A: Hier eine Anleitung wie Sprachversionen von Seiten gelöscht werden können: Den Content Editor starten (Ribbon-Menü > Campus > Backend):

1) Im Item-Tree die entsprechende Seite wählen und im Ribbon Menü zum Tab Versions wechseln.

2) Durch klicken auf den kleinen Sprachbutton ist ersichtlich, dass eine deutsche und eine englische Version der Seite vorhanden sind.

3) Durch klicken auf den grossen Sprachbutton zur löschenden Sprachversion wechseln (hier im Beispiel englisch).

4) Auf den Button Remove klicken.

5) Es erscheint ein Warnhinweis, ob die Version entfernt werden soll. Wenn Sie sicher sind, auf OK klicken.

6) Durch klicken auf den kleinen Sprachbutton ist ersichtlich, dass die englische Version gelöscht wurde. Bei Englsh:Englisch steht 0 versions.
Q: Auch nachdem ich alle Teaser und Texte auf der englischen Seite übersetzt habe, erscheint der Hinweis auf nur auf Deutsch verfügbare Inhalte. Wie finde ich die Ursache?
A: Falls alle Texte auf englisch übersetzt wurden, und die Meldung 'Some Content is only available in German' immer noch erscheint, liegt es daran, dass es von mindestens einem Element auf der Seite noch keine Englische Version gibt. Damit die Meldung verschwindet muss von jedem Element, auch Bilder und PDFs, eine englische Version erstellt werden. Im Backend kannst du jedes Element durchgehen und so leicht nachschauen, welche Items noch keine englische Version besitzt.
Oftmals ist es der Kontaktteaser. Es sieht auf englisch dann korrekt aus, weil Sitecore automatisch die korrekte Berufsbezeichnung übernimmt, Sitecore merkt aber auch, dass nichts angepasst wurde und geht dann davon aus, dass es noch deutsch ist. Es muss dann einfach eine englische Version davon im Backend angelegt werden. 
Die Meldung wird auch oft durch PDFs ausgelöst, die nur in einer Sprachversion vorhanden sind. Korrekt wuerde man ein PDF mit nur deutschen Infos hochladen und dann von demselben PDF-Item noch eine englische Version erstellen und das in der englischen Seitenversion anhängen. 
Bitte auch die Alt-Texte der Bilder kontrollieren.  
[Hier](https://applsupport.hslu.ch/webhelp/hm_sprachversion.htm?zoom_highlightsub=englische+version) ist beschrieben, wie eine englische Version eines Elements erstellt werden kann.

Q: Auf unserer Teamseite möchte ich die Teaser dichter zusammenbringen. Wie eliminiere ich die Leerräume?
A: Manchmal kann es dazu kommen, dass Elemente nicht an der richtigen Stelle platziert werden. Werden z.B. mehrere Teaser platziert, werden diese automatisch angeordnet. Dies kann aber dazu führen, dass das Layout nicht wie gewünscht aussieht. Durch Einfügen eines geschützten Umbruch, kann das Layout strukturiert werden. So können unnötige Lücken vermieden werden oder einzelne Elemente linksbündig oder rechtsbündig platziert werden.
Wie das geht ist [hier](http://applsupport.hslu.ch/web12/index.html?hm_geschuetzten_umbruch_einfuegen.htm) beschrieben.

In diesem Webhelp Artikel ist beschrieben, wie eine Englische Seite angelegt werden kann. Ausserdem ist das Shared Prinzip erklärt, welches Beschriebt, dass die Englsiche und deutsche Seite immer die gleichen Elemente besitzten müssen. Laut Webgovernance ist es nicht gewünscht nur englsiche Seiten zu erstellen. Technisch ist dies auch nicht möglich nur eine englische Version einer Seite anzulegen. Da die deutsche Seite voran geht, überschreibt die deutsche Seite immer die englische. Weiter sind in diesem Artikel ein paar Tipps aufgelistet, wie mit Sprachversion gearbeitet werden sollte.

HINWEIS: Zu beachten sind auch die Sprachhinweise DE / EN (https://applsupport.hslu.ch/webhelp/hm_sprachhinweise_de___en.htm).

Der Versions-Tab des Ribbon-Menüs bietet folgende Möglichkeiten:
 
Anlegen einer neuen Sprachversion via Language-Button. Anschliessend die Seite (ctrl+s) speichern.
Wechsel zwischen zwei bereits angelegten Sprachversionen.
Anlegen einer neuen Seitenversion
Löschen einer Seitenversion

Neue Sprachversionen einer Seite immer im Experience-Editor zu erstellen. So werden die eingefügten Elemente der deutschen Version auch in die englische Version übernommen und können anschliessend übersetzt werden.
Wird aber eine neue Sprachversion im Content-Editor erstellt, werden keine Elemente von der deutschen Version in der englischen Versionsseite erstellt!

Sitecore ist nach dem shared-Prinzip eingestellt. Wurde z.B. auf der englischen Seiten-Version ein Element (Item) nicht übersetzt, wird dieses Element auf der englischen Seite als deutsche Version angezeigt. Auch inhaltliche Änderungen beim deutschen Item werden dynamisch auf der englischen Seite dargestellt. Wir sprechen hier von einem Fallback. Dieser Fallback-Mechanismus existiert nur auf den englischen Seiten.
 
Hinweis: Sind von einer Seite zwei Sprachversionen vorhanden, muss die URL in beiden Sprachen angepasst werden!

Achtung: Sind Seitenversionen vorhanden, wird das Layout der Items von der aktuellsten deutschen Version auf die aktuellste englische Version kopiert. Der Kopiervorgang erfolgt jeweils beim speichern der aktuellen deutschen Version.Teaser 1, nur in deutscher Version vorhanden > "Fallback" funktioniert:

PRÄZISE ANTWORT:

====== 

RESPONSE:

Um die englische Sprachversion einer Seite zu löschen, gehen Sie folgendermaßen vor:

1. Öffnen Sie den Content Editor (Ribbon-Menü > Campus > Backend).

2. Wählen Sie im Item-Tree die entsprechende Seite aus und wechseln Sie im Ribbon Menü zum Tab Versions.

3. Klicken Sie auf den kleinen Sprachbutton, um die verfügbaren Sprachversionen anzuzeigen.

4. Klicken Sie auf den großen Sprachbutton zur zu löschenden Sprachversion (zum Beispiel Englisch).

5. Klicken Sie auf den Button "Remove".

6. Bestätigen Sie die Warnhinweise und klicken Sie auf "OK".

Nachdem Sie diese Schritte ausgeführt haben, sollte die englische Version der Seite gelöscht sein. Weitere Informationen finden Sie in diesem [Webhelp-Artikel](https://applsupport.hslu.ch/webhelp/hm_sprachversion.htm).

Bitte beachten Sie, dass von jeder Seite zwei Sprachversionen vorhanden sein müssen, daher ist es nicht empfohlen, nur englische Seiten zu erstellen. Wenn Sie weitere Unterstützung benötigen, wenden Sie sich bitte an den Websupport unter [websuport@hslu.ch](mailto:websuport@hslu.ch).

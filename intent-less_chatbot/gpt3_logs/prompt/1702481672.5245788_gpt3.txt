Im Experience Editor kann auf der Seite über die den Befehl "Campus > Publish Options" im Ribbon-Menü kontrolliert werden, ob und wann Inhalte publiziert werden.
Im Feld "Publish" definiert die Redaktion, wann ein nicht veröffentlichter Inhalt publiziert wird. Achtung: Die Seite wird immer auf den nächstfolgenden Veröffentlichungszeitpunkt live geschalten (entweder um 12:05 oder 00:05).

Im Feld "Unpublish" definiert die Redaktion, wann ein veröffentlichter Inhalt zurückgezogen wird.

Ist die Checkbox "Never publish" angekreuzt, wird ein Inhalt nicht publiziert.
Publish / Unpublish kann auch bei einzelnen Teaser oder auf Files in der Media Library angewandt werden.

Publish / Unpublish kann auch bei einzelnen Teaser oder auf Files angewandt werden

Wenn ein Teaser in der deutschen Version gelöscht wird, wird er auch in der englischen gelöscht. Es gilt die Regel, keine Inhalte nur in englisch, da dies fuer SEO schädlich ist.

Q: Gibt es eine Möglichkeit zu sehen, welche Teile meiner englischsprachigen Webseite angeblich nicht übersetzt sind, da der Hinweis auf nicht verfügbare deutsche Inhalte weiterhin erscheint?

Die Meldung wird auch oft durch PDFs ausgelöst, die nur in einer Sprachversion vorhanden sind. Korrekt wuerde man ein PDF mit nur deutschen Infos hochladen und dann von demselben PDF-Item noch eine englische Version erstellen und das in der englischen Seitenversion anhängen.

Q: Kann ich ein benutzerdefiniertes Bild hinterlegen, das erscheint, wenn ich eine Website auf Social Media verlinke?
A: Wenn kein Open Graph Bild auf der jeweiligen Seite hinterlegt ist, wird automatisch das Standardbild angezeigt. Um ein benutzerdefiniertes Bild anzuzeigen kann wie folgt vorgegangen werden: 
1) Im Content-Tree zur entsprechenden Seite navigieren.
2) Rechts im Contentbereich unter Spezial Meta Data können die Meta-Daten bearbeitet werden.
3) Open Graph Image - Durch Klicken auf Browse kann hier ein Bild aus der Media-Library ausgewählt werden, welches dann als Open Graph Image verwendet wird. Das Bild ist idealerweise 1200 x 630 Pixel gross oder Bildverhältnis 1.91:1. Maximum 1500px x 1500px oder 5 MB.
4) Checkbox Open Graph Image anzeigen - Hier kann gewählt werden, ob das Bild angezeigt werden soll oder nicht. 
Weitere Infos dazu findest du [hier](https://applsupport.hslu.ch/webhelp/hm_special_meta_data.htm?zoom_highlightsub=open+graph+bild)
 
 Evtl. musst du nach dem Einfuegen des Bildes den Link zu deiner Seite einmalig fuer die SoMe Plattformen bereinigen mit den folgenden Links:
 Facebook: https://developers.facebook.com/tools/debug/
 Twitter: https://cards-dev.twitter.com/validator
 LinkedIn: https://www.linkedin.com/post-inspector/
 Danach sollte dein Bild angezeigt werden.

WEBHELP-ARTIKEL 
 Key: Virtuelle Navigation (Link in der Navigation)
Navigation:  Navigation

Dieser Webhelp-Artikel beschriebt, was eine virtuelle Navigation ist. Mit einer virtuellen navigation können Seiten in der Navigation verlinkt werden die nicht direkte Unterseiten sind. So kann eine benutzerdefinierte Navigation erstellt werden. 

Eine Virtuelle Navigation (Navigation Virtual) ist ein Link in der Navigation und fügt in der Navigation und in der Seitenstruktur einen Verweis ein. Die Redaktion bestimmt, wohin dieser Verweis zeigt.
 
Achtung: Die Virtuelle Navigation funktioniert nur für interne Seiten (Insert Link).
 
Achtung: Wird ein Virtuelle Navigationseintrag eingefügt, kann eine Endlosschlaufe entstehen, welche das System zum Absturz bringt. Gefahr besteht, wenn:
 
1.Option "Nicht virtuelle Unterseiten anzeigen" ausgewählt ist
2.Eine der angezeigten Unterseiten wieder einen virtuellen Navigationseintrag enthält
3.Dieser Navigationseintrag auf einen Ort im Seitenbaum verweist, der über dem ursprünglichen Navigationseintrag liegt.
 
Achtung: Werden virtuelle Navigationseinträge dazu benutzt eine benutzerdefinierte Seitennavigation zu erstellen immer wie folgt konfigurieren:
1. "Anzeigen" aktiv
2. "Nicht virtuelle Unterseiten anzeigen" NICHT aktiv
3. "In Navigation Anzeigen" NICHT aktiv
4. "In Side-Navigation anzeigen" aktiv
 
 
Wie man eine virtuelle Navigationseintrag einfügen kann ist hier beschrieben: https://applsupport.hslu.ch/webhelp/index.html?hm_link_in_der_navigation_einfueg.htm.

In dieser Anleitung wird beschrieben, wie eine Seite verschoben werden kann. Ausserdem wird darauf hingewiesen, dass vor dem verschieben in manchen Fällen ein Redirect eingerichtet werden muss

1) Auf die zu verschiebende Seite navigieren und Im Ribbon-Menü den Befehl "Home > Move Page" aufrufen.
 
2) Im Dialog Zielseite anwählen und mit "Move" bestätigen.
 
Hinweis: Im Backend können Seiten auch per Drag & Drop sehr genau und einfach verschoben werden.
 
Achtung: Beim Verschieben einer Seite wird automatisch die Seiten-URL (inkl. aller Unterseiten!) geändert. Ohne Umleitung gelangen Webbesucher via Suchmaschinen wie Google oder auch über andere Werbemassnahmen auf Fehlerseiten («404: Seite nicht gefunden»), was verhindert werden sollte. Zudem werden die neuen bzw. geänderten URLs von Google wieder neu indexiert, was je nach Level unterschiedlich lange dauern kann.
 
Daher kann via Redirect (= Umleitung, im Voraus über den Web-Support beantragen) sichergestellt werden, dass die Webbesucher auf die richtige Seite umgeleitet werden und zudem auch das Google-Ranking der alten Seite auf die neue übertragen werden kann.

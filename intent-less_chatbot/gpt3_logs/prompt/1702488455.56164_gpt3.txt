Q: Ich hab schon wieder ein Frage, danke fuer Eure Hilfe:
Auf <<link>>
habe ich einen Link mit Anker definiert, 
 
Aber der Anker wird nicht angesprungen. Was habe ich falsch gemacht? 

A: Ein häufige Fehler bei Anker ist, dass das # fälschlicherweise entweder vergessen geht oder mitkopiert wird. Bei manchen Elementen braucht es das #, bei manchen nicht. 
- Für die Verlinkung innerhalb der gleichen Seite auf einen manuellen Teasern muss das # weggelassen werden. 
- Bei der Verlinkung von Rich Text Teasern auf der gleichen Seite, muss das # mitkopiert werden. 
- Bei der Verlinkung von externen Seiten muss das # immer mitkopiert werden. 
Hier sind die Webhelp-Artikel dazu:
https://applsupport.hslu.ch/webhelp/index.html?hm_anker_auf_anderer_seite.htm
https://applsupport.hslu.ch/webhelp/index.html?hm_anker_auf_anderer_seite.htm

Bei diesem Element können alle Daten des Kontakts manuell eingeben werden. Diese  Anpassungen werden aber nur für diese eine Seite gelten. Siehe hier: [https://applsupport.hslu.ch/webhelp/hm_kontaktperson-element_manuell_.htm?zoom_highlightsub=kontaktperson](https://applsupport.hslu.ch/webhelp/hm_kontaktperson-element_manuell_.htm?zoom_highlightsub=kontaktperson)
Q: Uns ist soebenaufgefallen, dass das untenstehende Bild in der Mobile Ansicht nicht angezeigt wird. 
<<link>>
Könnt ihr mir diesbezueglich weiterhelfen? 
 
A: Für mobile und Desktop sind unterschiedliche Moodboards zugewiesen. Die sind nicht automatisch gleich und müssen bei mobile und bei Desktop separat zugewiesen bzw. entfernt werden. Wird das Moodboard in der mobilen Version falsch oder gar nicht angezeigt, liegt es oft daran, dass kein mobiles Moodboard erstellt wurde oder dieses nicht vollständig konfiguriert wurde. 
1. In der Media Library muss ein seperates Bild für die Desktop und MoileMoodboardversion hochgeladen werden. 
2. Beim Zuweisen von Bild zum Moodboard, muss bei mobile das mobile Bild ausgwählt werden.
3. Bei den Page Options muss der Seite sowohl ein Desktop Moodboard als auch ein mobiles Moodboard zugewiesen werden. 
Hier die Anleitungslinks dazu:

Im Feld "Publish" definiert die Redaktion, wann ein nicht veröffentlichter Inhalt publiziert wird. Achtung: Die Seite wird immer auf den nächstfolgenden Veröffentlichungszeitpunkt live geschalten (entweder um 12:05 oder 00:05).

Ist die Checkbox "Never publish" angekreuzt, wird ein Inhalt nicht publiziert.

Q: Gestern hatte ich eine Info-Veranstaltung und plötzlich ist mir aufgefallen, dass am Nachmittag (möglicherweise auch schon am Vormittag), die Info-Veranstaltung auf der Produkt-Webseite fehlte. Dies, obwohl ich im Evento eingestellt habe, dass man sich bis heute anmelden kann. Unter den «Veranstaltungen» war der Anmeldelink jedoch noch zu finden. Wisst ihr an was das liegt? Wie kann ich sicherstellen, dass der Anmeldelink auch auf den Produkt-Webseiten sicher bis zum Zeitpunkt der Veranstaltung publiziert ist? 


A: Wenn ein Agendaeintrag nicht (mehr) im Call to Action Teaser auftaucht, könnte es an der "Start-Zeit-Problematik" liegen. Bei WeVe Anmeldungen kann es an den Einstellungen des Agendaeintrags liegen.
Start-Zeit-Problematik: Der Event bleibt nur solange in der Filterliste und im Call to Action Teaser, bis die End-Zeit erreicht ist. Ist keine End-Zeit eingestellt, dann bleibt der Event bis zur Startzeit des Ereignisses im Call to Action Teaser. Wurde sowohl keine Startzeit also auch keine Endzeit für den Event festgelegt, dann bleibt der Event bis 00:00h im Call to Action Teaser. Danach wird er weder in der Filterliste, noch im automatischen CtA-Teaser angezeigt (live ist er weiterhin).
Damit die Anmeldung über ONLA / WeVe funktioniert, muss die Evento ID in der Facts-Liste auf der Produktseite hinterlegt werden und beim Call to Action Teaser die Checkbox  "Kontextabhängige Informationen aus Evento anzeigen" aktiviert aktiviert sein (Standard)! Bei einer deaktivierten Checkbox, wird entsprechend kein Link bzw. Status Meldung angezeigt. Mehr dazu findest du [hier](https://applsupport.hslu.ch/webhelp/index.html?hm_links_der_linkliste_zuweisen.htm)

Q: Ich habe auf dieser Seite einen Image Teaser eingebaut und dann noch verlinkt mit einem A3 JPG. Leider geht das Pop-Up nicht auf.
Was habe ich vergessen? Konnte es in Web-Help nicht rausfinden.

A: Um ein Bild einzufügen, das beim Draufklicken grösser dargestellt wird, kann ein manueller Teaser oder eine Bildergeschichte (mehrere Bilder ) eingefügt werden. Beim Bearbeiten muss die Checkbox "Link (nur Bilder via Media Link) in Popup vergrössern" ausgewählt werden. Das vergrösserte Bild muss als Link ausgewählt werden.
Falls der Haken bei "Link (nur Bilder via Media Link) in Popup vergrössern" gesetzt wurde aber das Bild beim Draufklicken nicht grösser dargestellt wird, liegt es wohl daran, dass das Bild noch nicht in der grossen Version hochgeladen wurde. Wenn der Teaser dann bearbeitet wird, kann im Bereich  "Link" durch die Funktion  "Insert Media Link" das grössere Bild und die Checkbox "Link in Popup vergrössern" ausgewählt werden. Dadurch wird das grössere Bild beim Klick auf das Teaserbild angezeigt.

1.Der Ankername mit dem "Gartenzaun-Zeichen" # auf der Zielseite kopieren, z.B. #zwischentitel

4.Den Cursor im neuen Link platzieren und auf das Link-Symbol für externe Links anklicken.
 
5.Im Feld URL ganz am Schluss ohne Slash den Ankername anfügen z.B. xxxxxxx#zwischentitel

Hinweis: Unter dem Register Anchor (Anker) können im Rich-Text eigene Anker definiert werden. 

6.Die Seite speichern (ctrl-S)

Dieser Webhelp-Artikel enthält allgmeneine stylistische Tipps die beim erfassen von Texten auf der Website zu beachten sind.

Auszeichnung: Fett vor Kursiv: 
Oft empfiehlt es sich, Wichtiges, etwa Keywords, besonders hervorzuheben. Fettdruck ist der Kursivschrift vorzuziehen, weil Kursivschrift am Bildschirm schwerer zu lesen ist. Wir wenden Auszeichnungen punktuell an und nicht flächendeckend.

Links machen klar, wohin sie führen:
Ein Link verrät, wohin er führt. Richtig eingesetzt, dienen Links als Belege (ähnlich Zitaten in Forschungsarbeiten) und verleihen einem Text im Internet Autorität.
 
Beispiel: Es ist besser, so zum World Wide Web Konsortium zu verlinken als so: hier klicken.

Bildhafte Sprache erschwert das Übersetzen:
Texte im Netz werden nicht immer in der Sprache gelesen, in der sie ursprünglich geschrieben wurden. Google-Translate macht es möglich: Aus Deutsch wird Englisch. Je weniger idiomatische (das heisst sprach-typische Ausdrücke) verwendet werden, desto besser die automatischen Übersetzungen. Idiomatische Ausdrücke und Redewendungen sind deshalb durch nicht-bildliche Synonyme zu ersetzen.
 
Beispiel: „Sie ist gestorben“, statt „Sie hat das Zeitliche gesegnet.“

Es lohnt sich, Abkürzungen einzuführen:
In Texten für die Öffentlichkeit sind Abkürzungen bei der ersten Verwendung einzuführen. An der Hochschule Luzern gibt es ähnlich viele Abkürzungen wie im Schweizer Militär. Hier wie dort sind Abkürzungen – CCs, KAMs und so weiter – für Aussenstehende unverständlich.
 
Beispiel: „Das Competence Center Tourismus (CC Tourismus) hat vorgestern […] so berichtete Henriette Muster vom CC Tourismus.“

PRÄZISE ANTWORT:

====== 

RESPONSE:

Ein Anker in einer Webseite ist ein Sprunglink, der es ermöglicht, direkt zu einem bestimmten Abschnitt innerhalb derselben Seite zu gelangen. Dies kann durch das Setzen eines Ankers in einem Text und das Erstellen eines Links zu diesem Anker erreicht werden. Die Anker können in einem Manuellen Teaser oder in einem Rich-Text Teaser definiert werden. Zusätzlich können eigene Anker im Rich-Text Editor erstellt werden, die dann als Sprungmarke dienen. Bei Verlinkungen zu Ankerpunkten auf derselben Seite müssen die Ankername mit oder ohne "Gartenzaun-Zeichen" # kopiert und im Link eingefügt werden. Weitere Informationen zu Anker finden sich in diesem [Webhelp-Artikel](https://applsupport.hslu.ch/webhelp/hm_anker.htm) und in den Artikeln [Anker auf anderer Seite](https://applsupport.hslu.ch/webhelp/hm_anker_auf_anderer_seite.htm), [Eigener Anker im Rich-Text](https://applsupport.hslu.ch/webhelp/hm_eigener_anker_im_richtext_defi.htm) und [Anker auf gleicher Seite](https://applsupport.hslu.ch/webhelp/hm_anker_auf_gleicher_seite.htm).

CHAT-HISTORY: Human: Wie lösche ich die englische Sprachversion einer Seite, ohne die Deutsche zu löschen? 
AI: Um die englische Sprachversion einer Seite zu löschen, ohne die deutsche zu löschen, gehst du wie folgt vor:

1. Starte den Content Editor (Ribbon-Menü > Campus > Backend).
2. Wähle die entsprechende Seite im Item-Tree aus und wechsle im Ribbon-Menü zum Tab Versions.
3. Klicke auf den kleinen Sprachbutton, um zu sehen, ob eine deutsche und eine englische Version der Seite vorhanden sind.
4. Klicke auf den großen Sprachbutton zur zu löschenden Sprachversion (zum Beispiel englisch).
5. Klicke auf den Button "Remove".
6. Es erscheint ein Warnhinweis, ob die Version entfernt werden soll. Wenn du sicher bist, klicke auf "OK".
7. Durch Klicken auf den kleinen Sprachbutton kannst du überprüfen, ob die englische Version gelöscht wurde. Bei Englsh:Englisch sollte "0 versions" stehen.

Weitere Informationen und Anleitungen findest du in den folgenden Webhelp-Artikeln:
- [Sprach-Version](https://applsupport.hslu.ch/webhelp/hm_sprachversion.htm)
- [Seite EN Version erstellen](https://applsupport.hslu.ch/webhelp/hm_seite_uebersetzen.htm)

Bitte beachte, dass es wichtig ist, die Spracheinstellung beim Bearbeiten von Seiten im Experience Editor oder im Backend auf Deutsch einzustellen, um das unabsichtliche Erstellen einer englischen Version zu vermeiden.

Q: Wieso wird nicht mein hinterlegtes Bild auf Social Media dargestellt?
A: Wenn kein Open Graph Bild auf der jeweiligen Seite hinterlegt ist, wird automatisch das Standardbild angezeigt. Um ein benutzerdefiniertes Bild anzuzeigen kann wie folgt vorgegangen werden: 
1) Im Content-Tree zur entsprechenden Seite navigieren.

4) Checkbox Open Graph Image anzeigen - Hier kann gewählt werden, ob das Bild angezeigt werden soll oder nicht.

 Danach sollte dein Bild angezeigt werden.
 
Q: Warum wurde meine Seite nicht publiziert?
A: Wenn eine Seite nicht erscheint kann das an den Publish Einstellungen liegen: 
Im Experience Editor kann auf der Seite über die den Befehl "Campus > Publish Options" im Ribbon-Menü kontrolliert werden, ob und wann Inhalte publiziert werden.
Im Feld "Publish" definiert die Redaktion, wann ein nicht veröffentlichter Inhalt publiziert wird. Achtung: Die Seite wird immer auf den nächstfolgenden Veröffentlichungszeitpunkt live geschalten (entweder um 12:05 oder 00:05).

Im Feld "Unpublish" definiert die Redaktion, wann ein veröffentlichter Inhalt zurückgezogen wird.

Ist die Checkbox "Never publish" angekreuzt, wird ein Inhalt nicht publiziert.
Publish / Unpublish kann auch bei einzelnen Teaser oder auf Files in der Media Library angewandt werden.

Publish / Unpublish kann auch bei einzelnen Teaser oder auf Files angewandt werden

Ein weiterer Grun könnte an den Sprachversionen liegen. Seiten wie auch Seitenelemente müssen immer auf deutsch vorhanden sein. Wenn ein Teaser in der deutschen Version gelöscht wird, wird er auch in der englischen gelöscht. Es gilt die Regel, keine Inhalte nur in englisch, da dies fuer SEO schädlich ist.

Q: Ich habe soebendie Weiterbildungsseite der Informatik übersetzt und dabei habe ich gesehen, dass die Filterliste nur zwei WB-Angebote anzeigt. Ich kann es mir aktuell nicht erklären. Können wir das so rasch wie möglich bitte zusammen anschauen? Eigentlich sollten hier nämlich alle WB-Angebote aufgelistet sein, analog der deutschen Seite.
A: Bei Agendaeinträgen ist zu unterscheiden, ob die Seite offline ist, oder ob der Agendaeintrag einfach nicht in der Filterliste aufgelistet wird. Mögliche Ursachen dafür, dass eine Seite nicht öffentlich ist findest du [hier](https://applsupport.hslu.ch/webhelp/index.html?hm_publish_options.htm). Ausserdem kommt es bei  Agenda-Einträgen öfters vor, dass diese etwas länger haben, bis sie veröffentlicht werden.
Voraussetzung dafür, dass ein Event in einer Filterliste erscheint, ist, dass der Agendaeintrag live ist.  
Mögliche Ursachen dafür, dass ein Agendaeintrag live ist aber nicht der Filterliste aufgelistet wird : 
1. Der Event ist keiner Organisationseinheit (OE) zugeordnet
2. Falls es sich um eine Filterliste auf einer englischen Seite handelt, ist darauf zu achten, dass nur englische Seiten aufgeführt werden.
3. Start-Zeit-Problematik: Der Event bleibt nur solange in der Filterliste und im Call to Action Teaser, bis die End-Zeit erreicht ist. Ist keine End-Zeit eingestellt, dann bleibt der Event bis zur Startzeit in der Filterliste. Ist sowohl Startzeit als auch Endzeit leer, dann bleibt der Event bis 00:00h in der Filterliste und im Call to Action Teaser.
Spezifisch für Englische Seiten git: In englischen Filterlisten werden nur diejenigen Kurse angezeigt, die eine englische Version haben. Damit ein Event oder Kurs in der englischen Filterliste erscheint, muss die Seite des Events oder Kurses in einer englischen Version existieren.  Bitte keine englischen Versionen von Seiten anlegen, wenn diese nicht auch übersetzt werden, das wäre fuer die englische User Experience schlecht.
